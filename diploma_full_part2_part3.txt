2.1 Разработка структуры и архитектуры приложения
────────────────────────────────────────────────────

Веб-приложение «Qazaq IT Academy» представляет собой полнофункциональную образовательную платформу класса LMS (Learning Management System — система управления обучением) с интегрированным AI-помощником, многоуровневой геймификацией и встроенным интернет-магазином мерчандайза. Продукт ориентирован на организацию дистанционного и смешанного обучения IT-дисциплинам (программирование, веб-разработка, информатика, кибербезопасность, машинное обучение и другие направления) и реализован по клиент-серверной архитектуре: серверная часть предоставляет REST API, клиентская часть — интерактивный интерфейс в браузере.

Архитектура платформы построена по принципу модульности с чётким разделением серверной и клиентской частей. Связь между ними осуществляется исключительно через REST API, что обеспечивает:
  — независимую разработку и развёртывание каждой части;
  — возможность замены или добавления клиентов (мобильное приложение, сторонние интеграции) без изменения серверной логики;
  — чёткий контракт взаимодействия, задокументированный через OpenAPI/Swagger.


2.1.1 Технологический стек

СЕРВЕРНАЯ ЧАСТЬ (Backend):
  • Язык: Python 3.12+
  • Web-фреймворк: FastAPI — асинхронный высокопроизводительный фреймворк с автоматической генерацией OpenAPI-документации
  • ORM: SQLAlchemy — объектно-реляционное отображение для работы с базами данных
  • Валидация данных: Pydantic v2 (pydantic-settings) — строгая типизация и схемы запросов/ответов
  • Аутентификация: JWT (JSON Web Token) — создание токена с полями sub (user_id), email, role; токен действует 24 часа (настройка ACCESS_TOKEN_EXPIRE_MINUTES)
  • Хеширование паролей: passlib с bcrypt — необратимое хеширование для безопасного хранения
  • Rate Limiting: SlowAPI — ограничение частоты запросов (60 попыток входа в минуту, 10 регистраций, 5 заявок на покупку)
  • Интеграция с LLM: OpenAI API (модель gpt-4o-mini) И/ИЛИ Google Gemini API (модель gemini-2.0-flash) — выбор провайдера автоматически определяется по наличию ключей; приоритет отдаётся OpenAI
  • Генерация документов: Pillow (PIL) — рендеринг PNG-сертификатов из шаблона с персонализацией; openpyxl — экспорт Excel-отчётов
  • Планировщик задач: APScheduler — фоновая задача ежедневного начисления наград топ-5 студентов лидерборда (cron в 00:05 по часовому поясу Asia/Almaty + hourly catchup)
  • SMTP: smtplib — отправка HTML-писем с учётными данными и ссылками подтверждения покупки
  • HTTP-сервер: Uvicorn с горячей перезагрузкой --reload
  • Статические файлы: FastAPI StaticFiles – раздача каталога uploads (аватары, видео, сертификаты, вложения заданий, конспекты)

КЛИЕНТСКАЯ ЧАСТЬ (Frontend):
  • Фреймворк: Next.js 16 (App Router) — серверный рендеринг, маршрутизация, оптимизация
  • Библиотека UI: React 19 с TypeScript — типизированные компоненты
  • Стилизация: Tailwind CSS 4 — utility-first CSS с поддержкой светлой и тёмной тем
  • Серверное состояние: TanStack Query (React Query) — кэширование, инвалидация, фоновое обновление данных с API
  • Клиентское состояние: Zustand — глобальное хранилище аутентификации и пользователя с персистенцией в localStorage
  • Формы: react-hook-form + Zod — валидация на клиенте перед отправкой на сервер
  • Анимации: Motion (Framer Motion) — плавные переходы блоков, анимация появления виджетов
  • Rich Text: TipTap — визуальный редактор для преподавателей (форматирование инструкций заданий: жирный, курсив, списки, код, ссылки)
  • Графики: Recharts — визуализация аналитики (прогресс курсов, статистика, градация навыков)
  • HTTP-клиент: axios — запросы к API с автоматической подстановкой JWT в заголовок Authorization

ХРАНЕНИЕ ДАННЫХ:
  • Продакшен: PostgreSQL (сервис в Docker Compose) — полнофункциональная реляционная СУБД
  • Локальная разработка: SQLite (файл education.db) — переключение через переменную окружения DATABASE_URL
  • Миграции: автоматические «лёгкие» миграции при старте приложения (модуль run_migrations + множественные скрипты migrate_*.py для добавления столбцов и таблиц)
  • Файловое хранилище: каталог uploads/ с подкаталогами:
    — uploads/videos/ — видеоматериалы уроков
    — uploads/certificates/ — шаблон сертификата
    — uploads/certificates/issued/ — сгенерированные персональные сертификаты (cert_{id}.png)

ИНТЕРНАЦИОНАЛИЗАЦИЯ (i18n):

2.1.2 Ролевая модель

Система поддерживает шесть типов пользователей, каждый из которых обладает своим набором прав и разделов интерфейса:

1) АДМИНИСТРАТОР (admin)
   — Полный контроль над платформой: управление пользователями, курсами, тестами, магазином, аналитикой
   — Создание/редактирование/удаление пользователей всех ролей
   — Привязка родителей к детям (parent_id)
   — Управление заявками на курсы (одобрение/отклонение)
   — Управление заказами Q-Market и курьерами
   — Просмотр обращений в техподдержку
   — Поля модели: system_role (Суперадминистратор / Администратор факультета / Администратор кафедры), permissions (JSON-массив прав), can_create_users, can_delete_users, can_edit_courses, can_view_analytics, can_configure_system
   — Дополнительные поля: education_level, email_personal, areas_of_responsibility

2) ДИРЕКТОР (director)
   — Те же права, что и у администратора: доступ к админ-панели, аналитике, управлению пользователями и курсами
   — Получает уведомления о новых заявках и обращениях в поддержку

3) КУРАТОР (curator)
   — Доступ к административной панели с ограничениями
   — Специальные поля: curated_courses (JSON-мфессив курируемых курсов), consultation_schedule (JSON-расписание консультаций), consultation_location
   — Права: can_view_performance, can_message_students, can_view_attendance, can_call_parent_teacher_meetings, can_create_group_announcements
   — Получает уведомления о заявках студентов

4) ПРЕПОДАВАТЕЛЬ (teacher)
   — Управление учебными группами (TeacherGroup) и их студентами (GroupStudent)
   — Создание заданий с Rich Text, рубриками оценивания, вложениями (файлы, ссылки, видео)
   — Проверка сданных работ студентов: выставление оценки, комментарии (публичные и приватные)
   — Создание материалов и вопросов (TeacherMaterial, TeacherQuestion) для каждой темы курса
   — Управление лентой курса (CourseFeedPost)
   — Приглашение других преподавателей в группу (multi-teacher)
   — Специальные поля: gender, identity_card, iin, subjects_taught, student_counts, hire_date, employment_status, teaching_hours, academic_interests, department, position

5) СТУДЕНТ (student)
   — Дашборд с персонализированной статистикой и мотивацией
   — Запись и прохождение курсов: видео, теория, конспекты, задания, тесты
   — Участие в AI Challenge (соревнование с ИИ)
   — Общение через AI-помощника
   — Покупки в Q-Market за внутриигровую валюту (Q-coins)
   — Участие в сообществе, ведение заметок к темам
   — Обращения в техподдержку
   — Поля: points (баланс Q-coins), is_premium (статус Premium-подписки)

6) РОДИТЕЛЬ (parent)
   — Мониторинг прогресса привязанных детей (связь через parent_id)
   — Просмотр успеваемости по курсам, оценок за задания, сертификатов
   — Сравнение позиций детей в общем рейтинге
   — Экспорт отчётов в Excel
   — Специальные поля: work_place, kinship_degree (Отец/Мать/Опекун), educational_process_role (Законный представитель/Опекун), education, academic_degree, email_work, phone_work, office, reception_hours

На сервере каждый API-эндпоинт защищён middleware get_current_user (извлечение JWT из заголовка Authorization: Bearer) и дополнительными проверками роли, что исключает обход ограничений только средствами клиентского интерфейса.

Серверная часть организована в 27 маршрутных модулей (routers), каждый из которых отвечает за свою предметную область:

  /api/courses — каталог курсов, записи, детали
  /api/topics — темы курсов, контент, видео, theory
  /api/tests — тесты и вопросы, прохождение, результаты
  /api/progress — прогресс студента, отметка пройденных тем
  /api/ai — AI-помощник (чат)
  /api/challenge — AI Challenge (соревнование с ИИ)
  /api/notifications — система уведомлений
  /api/schedule — расписание и календарь
  /api/teacher — управление группами, задания, проверка работ, материалы, вопросы, лента курса
  /api/analytics — аналитика для администраторов
  /api/analytics_student — аналитика для студентов (сравнение, прогнозы, дерево навыков)
  /api/parent — родительская панель (прогресс детей, отчёты, рейтинг)
  /api/payments — платежи, подтверждение оплаты
  /api/assignments — задания (со стороны студента)
  /api/private_comments — приватные комментарии к заданиям и материалам
  /api/shop — Q-Market (каталог, корзина, покупки, избранное)
  /api/dashboard — агрегированные данные для дашборда

  /api/reviews — отзывы о курсах
  /api/community — сообщество (публикации, лайки)
  /api/questions — вопросы преподавателя к темам
  /api/support — техподдержка (обращения студентов)

Каждый модуль содержит Pydantic-схемы для валидации входящих данных и формирования ответов, что обеспечивает строгую типизацию на уровне контракта API.


2.2 Реализация системы аутентификации и авторизации
────────────────────────────────────────────────────

2.2.1 Серверная реализация

Модуль аутентификации реализован в файле app/api/routes/auth.py и базируется на протоколе OAuth2 с использованием JWT. Основная логика входа представлена на Листинге 1.

[ЛИСТИНГ 1: Реализация эндпоинта аутентификации (FastAPI)]
Путь к файлу: /home/nurjaks/Dev/LMS platform - order/backend/app/api/routes/auth.py (строки 36-50)
```python
@router.post("/login", response_model=Token)
@limiter.limit("60/minute")
def login(request: Request, data: UserLogin, db: Session = Depends(get_db)):
    # 1. Нормализация данных и поиск пользователя
    email = (data.email or "").strip().lower()
    user = db.query(User).filter(User.email == email).first()
    
    # 2. Верификация пароля (bcrypt)
    if not user or not verify_password(data.password, user.password_hash):
        raise HTTPException(status_code=401, detail="Неверный email или пароль")
    
    # 3. Регистрация активности и создание JWT
    log_activity(db, user.id, "login", "user", user.id)
    token = create_access_token(data={
        "sub": str(user.id), 
        "email": user.email, 
        "role": user.role
    })
    return Token(access_token=token)
```

[СКРИНШОТ КОДА 1: Эндпоинт Login в auth.py]

В приведенном коде (Листинг 1) видно применение декоратора `@limiter`, который предотвращает атаки методом перебора (Brute-force). Пароль проверяется против хешированного значения, хранящегося в БД, что обеспечивает безопасность данных даже при утечке базы.

POST /api/auth/login — вход в систему:
  • Принимает email и password (схема UserLogin с валидацией Pydantic)
  • Email нормализуется: strip().lower() + автокоррекция типичных опечаток (словарь _LOGIN_EMAIL_TYPOS)
  • Поиск пользователя в БД по email
  • Проверка пароля через verify_password(data.password, user.password_hash) — bcrypt-сравнение
  • При неудаче — HTTP 401 с сообщением «Неверный email или пароль»
  • При успехе — запись в журнал активности (log_activity), создание JWT-токена с payload: {sub: user_id, email, role}
  • Возврат токена в формате {access_token: "...", token_type: "bearer"}
  • Rate Limiting: 60 запросов в минуту с одного IP (настройка AUTH_LOGIN_RATELIMIT)

POST /api/auth/register — регистрация:
  • Эндпоинт существует, но в текущей конфигурации отключён: вызывает HTTP 403 «Регистрация временно отключена»
  • Новые пользователи создаются через админ-панель или автоматически при подаче заявки/оплате курса
  • Rate Limiting: 10 запросов в минуту

Безопасность:
  • Пароли хранятся исключительно как bcrypt-хеши (поле password_hash в таблице users)
  • JWT-токен содержит минимальный payload (user_id, email, role) без чувствительных данных
  • Время жизни токена: 24 часа (ACCESS_TOKEN_EXPIRE_MINUTES = 1440)
  • Механизм зависимости get_current_user: извлекает токен из заголовка Authorization, декодирует JWT, загружает пользователя из БД — обеспечивает единообразную аутентификацию для всех защищённых эндпоинтов
  • Отдельный хелпер get_current_admin_user: дополнительно проверяет роль пользователя (admin, director, curator)

2.7.1 Управление группами

TeacherGroup — учебная группа — центральная сущность, связывающая преподавателя, курс и студентов.

Модель TeacherGroup:
  — id, teacher_id, course_id, group_name, description, max_students, is_active, created_at

Функционал учебных групп:
  • Создание группы: выбор курса, название, описание, лимит студентов

  • Удаление студентов из группы

  • Просмотр списка студентов группы с их прогрессом

2.7.2 Система заданий (Assignments)

TeacherAssignment — задание преподавателя — реализует полноценную функциональность Google Classroom:

Модель TeacherAssignment:
  — teacher_id, group_id, course_id, topic_id — привязка к группе, курсу, теме
  — title, description — название и описание (поддерживает HTML через Rich Text)
  — deadline — крайний срок сдачи
  — max_points — максимальный балл (по умолчанию 100)
  — attachment_urls (JSON) — вложенные файлы (загруженные на сервер)
  — attachment_links (JSON) — ссылки на внешние ресурсы
  — video_urls (JSON) — видео (загруженные или YouTube/Vimeo)
  — test_id — привязка к тесту (опционально)
  — closed_at — ручное закрытие задания преподавателем
  — reject_submissions_after_deadline — автоматический отказ после дедлайна
  — is_synopsis — пометка «конспект» (специальный тип задания для потока обучения)
  — allow_student_class_comments — разрешать комментарии студентов
  — allow_student_edit_submission — разрешать редактирование сданной работы

Создание задания (CreateAssignmentFullPageModal, ≈126 КБ):
  • Полноэкранное модальное окно в стиле Google Classroom

```python
def video_requirement_met(db: Session, user_id: int, topic: CourseTopic) -> bool:
    # 1. Если видео нет, условие выполнено
    if not (topic.video_url or "").strip():
        return True
    
    # 2. Поиск прогресса пользователя
    prog = db.query(StudentProgress).filter(
        StudentProgress.user_id == user_id,
        StudentProgress.topic_id == topic.id
    ).first()
    
    # 3. Проверка: просмотрено ли более 90% видео
    effective_duration = (topic.video_duration or 0) or 300
    watched = (prog.video_watched_seconds if prog else 0) or 0
    return watched / effective_duration >= 0.90
```

```typescript
useEffect(() => {
  // 1. Мониторинг фокуса окна
  const handleBlur = () => {
    setIsCheatingWarning(true);
    logSuspiciousActivity("tab_switch");
  };

  // 2. Мониторинг полноэкранного режима
  const handleFullscreenChange = () => {
    if (!document.fullscreenElement) {
      setIsCheatingWarning(true);
      logSuspiciousActivity("exit_fullscreen");
    }
  };

  document.addEventListener("fullscreenchange", handleFullscreenChange);
  
  return () => {
    window.removeEventListener("blur", handleBlur);
    document.removeEventListener("fullscreenchange", handleFullscreenChange);
  };
}, []);
```

Логика, представленная на Листинге 4, позволяет мгновенно реагировать на попытки студента переключить вкладку или выйти из полноэкранного режима для поиска ответов. При обнаружении таких действий система блокирует тест и выводит предупреждение (Рисунок 4).

Расположение на сайте: Прохождение теста (в любой теме) -> Выход из полноэкранного режима (ESC) или переключение вкладки (Alt+Tab)

1. Блокировка навигации и выхода:

3. Защита контента (Content Lockdown):
   — Блокировка контекстного меню (onContextMenu): запрет на использование правой кнопки мыши.
   — Запрет выделения текста (selectstart): студент не может выделить текст вопроса для копирования.
   — Блокировка горячих клавиш: через обработчик keydown запрещены комбинации Ctrl+C (копирование), Ctrl+V (вставка), Ctrl+X (вырезание), Ctrl+A (выделить всё).

4. Интеграция с ИИ:
   — Во время активного теста AI-помощник переходит в строгий режим детекции контекста (is_test_context=True), автоматически отклоняя любые вопросы, содержащие фрагменты заданий или просьбы о помощи в решении.

```python
    ai_answers = solve_quiz_questions(questions, body.level)
    
    return ChallengeStartResponse(
        questions=questions,
        ai_responses=ai_answers,
        time_limit=ROUND_TIME_LIMIT
    )
```

2.10.5 Механика соревнования

Старт (POST /api/challenge/start):
  1. Проверка: студент должен пройти хотя бы 1 тему в курсе трека
  2. Выбор вопросов: случайная выборка из тестов пройденных тем (ограничение по уровню)
  3. Вопросы отправляются LLM для получения ответов ИИ (solve_quiz_questions)
  4. Кэширование: ответы ИИ сохраняются в AIChallengeCache (хеш вопросов + уровень + режим) для повторного использования
  5. Возврат: вопросы, времена ответов ИИ, лимит раунда

Отправка результатов (POST /api/challenge/{id}/submit):
  1. Подсчёт: правильные ответы студента и ИИ
  2. Бонусные очки за серии правильных ответов: 2 подряд = +1, 3 подряд = +2, 4 подряд = +3, 5 подряд = +4
  3. Определение победителя по сумме очков

  5. Начисление Q-coins победителю
  6. Создание уведомления с результатами
  7. Сохранение в таблицу AIChallenge (user_id, course_id, score, max_score, ai_score, ai_level)
  8. AI-рекомендации по ошибочным темам (get_challenge_recommendations через LLM)

Встроенные вопросы (LOCAL_CLASSIC_TRACK_QUESTIONS):

```python
def add_coins(db: Session, user_id: int, amount: int, reason: str, *, apply_premium_multiplier: bool = True) -> bool:
    # 1. Поиск пользователя и проверка статуса Premium
    user = db.query(User).filter(User.id == user_id).first()
    is_premium = getattr(user, "is_premium", 0) == 1
    
    # 2. Применение Double Coins для Premium
    actual_amount = amount
    if apply_premium_multiplier and is_premium:
        actual_amount = amount * 2
    
    # 3. Начисление, логирование и создание уведомления
    user.points = (user.points or 0) + actual_amount
    db.add(CoinTransactionLog(user_id=user_id, amount=actual_amount, reason=reason))
    db.add(Notification(user_id=user_id, title="Монеты начислены", message=f"+{actual_amount} coins"))
    return True
```

2.16.1 Логика формирования Skill Tree

Визуализация навыков студента (Skill Tree) строится на основе агрегации оценок за тесты по каждой компетенции. Техническая реализация сбора данных для дерева навыков представлена на Листинге 12.

[ЛИСТИНГ 12: Агрегация данных для дерева навыков (analytics_student.py)]
Путь к файлу: /home/nurjaks/Dev/LMS platform - order/backend/app/api/routes/dashboard.py (строки 40-73 — используется для базовой аналитики)
```python
def get_skill_progression(db: Session, user_id: int):
    # 1. Сбор всех завершенных тем
    progress = db.query(StudentProgress).filter(
        StudentProgress.user_id == user_id,
        StudentProgress.is_completed == True
    ).all()
    
    # 2. Группировка по категориям (компетенциям)
    skills = {}
    for p in progress:
        category = p.course.category.name
        skills[category] = skills.get(category, 0) + (p.test_score or 0)
        
    # 3. Нормализация для визуализации
    return {k: min(v / 10, 100) for k, v in skills.items()}
```

2.14 Глубокая аналитика обучения и мониторинга
─────────────────────────────────────────────

Платформа предоставляет детализированную аналитику как для администраторов, так и для студентов (модуль analytics_student.py):

   — Learning Activity Sources: Агрегация активности по 4 категориям: уроки (пройденные темы), задания (проверенные), тесты, сертификаты. Система сравнивает показатели текущего периода (7/30 дней) с предыдущим и рассчитывает процент роста/падения.
   — Weekly/Monthly Activity: Визуализация ежедневной активности в формате «сетки активности» (аналог GitHub contributions) с тремя уровнями интенсивности (low/medium/high).
   — Performance Metric: Детальное отслеживание времени просмотра видео по дням и среднего балла за тесты.
   — Premium Analytics: Для Premium-пользователей доступен глубокий архив данных (30 дней), для обычных — последние 2 дня, что стимулирует переход на платную подписку.

2.15 Система достижений и ежедневных квестов
─────────────────────────────────────────────

Мобильная геймификация реализована через механизмы удержания (Retention):

1. Ежедневные квесты (Daily Quests):
   — Login: награда за ежедневный вход (стак серий до 7 дней)
   — Quiz Master: награда за успешную сдачу 1 теста в день
   — Theory Watcher: награда за просмотр видео-урока
   — Topic Completer: награда за завершение полной темы (видео+задание+тест)
   — Система Claim: награда не начисляется автоматически, студент должен «забрать» её в дашборде.

2. Система достижений (Achievements):
   — «Первая победа» (First Win): за завершение первого курса
   — «Неделя активности» (Activity Week): за 7 дней непрерывного обучения
   — «Отличник» (Excellent): за 100% результат в тесте
   — «Мастер практики» (Practice Master): за 50 проверенных домашних заданий
   — Каждое достижение имеет уникальную иконку, дату получения и цветную индикаторную рамку.

2.16 Монетизация: Premium-подписка и ограничения
──────────────────────────────────────────────────

Для коммерциализации платформы реализована система Premium-доступа (модули premium.py и progress.py):

1. Преимущества Premium:
   — Снятие лимита на просмотр видео (у обычных пользователей — 2 часа в сутки)
   — Скидка 15% на все товары в Q-Market
   — Множитель x2 при начислении Q-coins за обучение
   — Доступ к персональным заметкам (Topic Notes) в каждой теме
   — Расширенная историческая аналитика (30 дней вместо 2)
   — Доступ к эксклюзивным Premium-курсам

2. Техническая реализация лимитов:
   — Использование функции check_daily_video_watch_limit: сервер подсчитывает время всех просмотров за текущие сутки (UTC+6) и блокирует обновление прогресса при достижении 7200 секунд (2 часа) для Free-аккаунтов.

  — Запустит backend (Uvicorn на порту 8000)
  — Установит Node.js-зависимости (npm install)
  — Запустит frontend (Next.js на порту 3000)

4. Открыть в браузере:

   — Swagger API: http://127.0.0.1:8000/docs

5. Остановка: Ctrl+C в том же окне

3.2.1 Главная страница

1. Откройте http://localhost:3000 в браузере.
2. Отображается маркетинговая главная страница с секциями: Hero-блок, статистика, преимущества, технологии, каталог курсов, отзывы, FAQ, контакты.
3. В шапке сайта:
   — Переключатель языка: нажмите «RU» / «KZ» / «EN» для смены языка. Выбор сохраняется.
   — Переключатель темы: иконка солнца/луны для переключения светлой/тёмной темы.
   — Кнопка «Жеке кабинет» (Личный кабинет) — переход к авторизации.
4. Прокрутите вниз для просмотра каталога курсов. Нажмите на фильтр категории («Программирование», «Веб», «Data Science», «IT» или «Все»).
5. Карточка курса: нажмите «Сатып алу» (Купить) для перехода к форме покупки.
6. Внизу страницы доступен виджет «Подберите курс» — интерактивная анкета, помогающая определить подходящий курс.


3.2.2 Покупка курса (для новых пользователей)

1. На главной странице нажмите «Купить» на карточке курса.
2. Заполните форму:
   — Email студента (будет использоваться как логин)
   — ФИО студента
   — Телефон, город, дата рождения, ИИН (опционально)
   — Email родителя, ФИО родителя, телефон (опционально — для несовершеннолетних)
   — Способ оплаты (Kaspi, Halyk, карта и др.)
3. Нажмите «Оплатить».
4. На указанный email придёт письмо с:
   — Временным логином и паролем для студента
   — Временным логином и паролем для родителя (если указан)
   — Ссылкой для подтверждения покупки
5. Перейдите по ссылке из письма для подтверждения.
6. На странице подтверждения отобразятся новые учётные данные. Сохраните их.
7. Перейдите на страницу входа и войдите в систему.


3.2.3 Авторизация

1. Перейдите на страницу входа: нажмите «Жеке кабинет» в шапке.
2. Введите email (логин) и пароль.
3. При необходимости нажмите иконку «глаз» для отображения пароля.
4. Нажмите «Кіру» (Войти).
5. При ошибке отобразится сообщение «Неверный email или пароль».
6. После успешного входа откроется дашборд, соответствующий вашей роли.

Тестовые учётные записи:
  admin@edu.kz / admin123 — Администратор
  director@edu.kz / director123 — Директор
  curator@edu.kz / curator123 — Куратор
  teacher1@edu.kz / teacher123 — Преподаватель
  parent@edu.kz / parent123 — Родитель
  student1@edu.kz / student123 — Студент


3.2.4 Навигация в приложении

На широком экране (десктоп):
  — Боковое меню (слева): дашборд, курсы, задания, расписание, лидерборд, сообщество, магазин, AI, аналитика, профиль и другие разделы (зависят от роли).
  — Верхняя панель: строка поиска, колокольчик уведомлений (badge с количеством непрочитанных), аватар (переход в профиль), язык, тема.

На узком экране (мобильные):
  — Нижняя панель навигации: 5 ключевых разделов с иконками.
  — Верхняя панель: логотип, уведомления, аватар.

Переключение языка и темы доступно из любого экрана.


3.2.5 Дашборд студента

1. После входа как студент открывается дашборд.
2. Обзор виджетов:
   — Приветствие с именем пользователя.
   — Карточки статистики: курсы, баллы (Q-coins), прогресс, записи.
   — «Продолжить обучение»: нажмите, чтобы перейти к последней незавершённой теме.
   — Ежедневные задания: отслеживайте прогресс (просмотр видео, ответы на вопросы, конспект). При выполнении нажмите кнопку получения награды.
   — Мотивация: ваша позиция в рейтинге и ближайшие конкуренты.
   — AI Challenge: нажмите для начала соревнования с ИИ.
   — Предстоящие дедлайны: список ближайших заданий.
   — Сообщество: последние публикации.


3.2.6 Мои курсы и прохождение материала

1. В боковом меню выберите «Курстар» (Курсы).
2. Отобразится список курсов, на которые вы записаны.
3. Нажмите на карточку курса для перехода к странице курса.
4. На странице курса:
   — Список тем с индикаторами: ✅ пройдено, 🔓 доступно, 🔒 заблокировано.
   — Визуализация пути обучения (Learning Path) — дорожка прогресса.
5. Нажмите на доступную тему.
6. Порядок прохождения темы:
   а) ВИДЕО: досмотрите видео до конца (≥90%). Перемотка вперёд ограничена — нельзя перескочить непросмотренную часть. Доступна смена скорости (1x, 1.25x, 1.5x, 2x).
   б) ТЕОРИЯ: после просмотра видео разблокируется текстовый контент. Прочитайте и изучите теорию.
   в) ЗАМЕТКИ: ведите личные заметки в поле заметок. Они сохраняются автоматически.
   г) КОНСПЕКТ: загрузите файл конспекта (при наличии задания).
   д) ДОМАШНЕЕ ЗАДАНИЕ: выполните задание от преподавателя (загрузите файл + комментарий).
   е) ТЕСТ: после проверки задания преподавателем откроется тест. Пройдите его.


3.2.7 Прохождение теста

1. На странице темы нажмите «Тестті бастау» (Начать тест) — доступно только при выполнении предыдущих этапов.
2. Тест содержит 20 вопросов с 4 вариантами ответа каждый.
3. На каждом шаге:
   — Прочитайте вопрос.
   — Выберите один вариант ответа.
   — Нажмите «Келесі» (Далее) для перехода к следующему вопросу.
   — Кнопка «Артқа» (Назад) для возврата.
   — Прогресс-бар показывает количество отвеченных вопросов.
4. На последнем вопросе нажмите «Аяқтау» (Завершить).
5. Экран результатов:
   — Итоговый процент правильных ответов.
   — Разбор каждого вопроса: ✅ правильный / ❌ неправильный, ваш ответ vs правильный.
   — При успешном прохождении (выше порога) — начисление Q-coins и генерация сертификата.
6. Сертификат доступен для скачивания из профиля или раздела достижений.


3.2.8 Задания от преподавателя

1. В боковом меню выберите «Тапсырмалар» (Задания) или откройте задание из страницы курса.
2. Список заданий с фильтрацией: все, не сдано, на проверке, оценено.
3. Нажмите на задание для просмотра:
   — Описание задания (форматированный текст)
   — Вложения: файлы, ссылки, видео от преподавателя
   — Рубрика оценивания: критерии с максимальными баллами
   — Дедлайн (цветовая индикация срочности)
4. Сдача работы:
   — Загрузите файл с решением (перетаскивание или выбор)
   — Добавьте текстовый комментарий (опционально)
   — Нажмите «Жіберу» (Отправить)
5. После проверки:
   — Оценка и комментарий преподавателя
   — Баллы по каждому критерию рубрики
6. Публичные комментарии: общение с преподавателем и одногруппниками под заданием.
7. Приватные комментарии: видны только вам и преподавателю.


3.2.9 AI-помощник

1. Нажмите на плавающую кнопку чата (🤖) в правом нижнем углу экрана.
2. Откроется окно AI-чата.
3. Введите вопрос по теории или практике и нажмите «Отправить» / Enter.
4. AI ответит на языке вашего вопроса (RU/KZ/EN).
5. Что можно спрашивать:
   — Объяснение теоретических концепций
   — Разбор ошибок в вашем коде (вставьте код в сообщение)
   — Советы по обучению
   — Помощь с выбором курса
6. Ограничения:
   — AI НЕ даёт прямых ответов на тестовые вопросы.
   — AI НЕ решает задания полностью за вас.
   — При попытке получить готовый ответ на тест, AI выдаст предупреждение.
   — Во время прохождения теста AI дополнительно ограничен.
7. Учитывайте дневные лимиты (зависят от типа аккаунта: free/premium).


3.2.10 AI Challenge (Соревнование с ИИ)

1. В боковом меню выберите «AI Challenge» или нажмите на карточку AI Challenge на дашборде.
2. Выберите курс/трек: Python, Web, Информатика, Кибербезопасность.
3. Выберите уровень ИИ:
   — Начинающий — лёгкий соперник (4 вопроса, 90 секунд)
   — Средний — достойный соперник (7 вопросов, 130 секунд)
   — Эксперт — сильный соперник (12 вопросов, 150 секунд)
4. Выберите режим игры:
   — Классический (quiz) — стандартные вопросы с 4 вариантами
   — Flashcard — карточки с определениями
   — Memory — игра «найди пару»
   — Найди баг — поиск ошибки в коде
   — Угадай вывод — определение результата выполнения кода
   — Скоростной код — написание кода на время
5. Нажмите «Бастау» (Старт).
6. Отвечайте на вопросы. ИИ отвечает параллельно (его прогресс виден).
7. Таймер раунда тикает. Постарайтесь ответить правильно и быстро.
8. Бонусы за серии правильных ответов: 2 подряд = +1 балл, 3 = +2, 4 = +3, 5 = +4.
9. По завершении — экран результатов:
   — Ваши правильные / всего
   — Правильные ИИ / всего
   — Время: ваше vs ИИ
   — Кто победил
   — Начисленные Q-coins (до 200 за победу)
   — AI-рекомендации по темам, в которых вы ошиблись
Условие доступа: необходимо пройти хотя бы одну тему в курсе выбранного трека.


3.2.11 Q-Market (Магазин)

1. В боковом меню выберите «Q-Market» (Магазин).
2. Каталог товаров:
   — Фильтрация по категориям (книги, сувениры, техника и др.)
   — Каждая карточка: фото, название, описание, цена в Q-coins
   — Premium-пользователи видят скидочную цену (-15%)
3. Действия с товаром:
   — «В корзину» — добавить в корзину
   — «❤️» — добавить в избранное
   — «Купить» — купить сразу
4. Корзина:
   — Нажмите иконку корзины вверху или кнопку «Корзина»
   — Изменяйте количество (+/-)
   — Удаляйте товары
   — Просмотрите итоговую сумму
   — Нажмите «Оформить заказ»
5. Ввод адреса доставки: город, улица, дом, квартира.
6. Подтверждение покупки — анимация успешного заказа, ожидаемая дата доставки.
7. Мои покупки:
   — Статусы: ожидание, в пути, доставлено, отменено
   — Отмена заказа: возврат 50% Q-coins
   — Ожидаемая дата доставки: 6-7 дней
8. Баланс Q-coins отображается в шапке и на странице магазина.


3.2.12 Лидерборд

1. В боковом меню выберите «Көшбасшылар» (Лидерборд).
2. Подиум топ-3: золото, серебро, бронза с аватарами.
3. Таблица: место, имя, баллы, курсы, активность.
4. Ваша позиция выделена цветом.
5. Фильтр по курсу (опционально).


3.2.13 Расписание и календарь

1. В боковом меню выберите «Кесте» (Расписание).
2. Календарь текущего месяца с отмеченными днями событий.
3. Нажмите на день с событием для просмотра деталей.
4. Модальное окно с деталями: время, описание, ссылки.


3.2.14 Сообщество

1. В боковом меню выберите «Қоғамдастық» (Сообщество).
2. Лента публикаций:
   — Фильтр по тегам: «Стратегии», «Советы»
   — Каждая публикация: автор, текст, дата, лайки
   — Premium-бейдж у Premium-пользователей
3. Создание публикации:
   — Выберите тег
   — Введите текст (1-2000 символов)
   — Нажмите «Жариялау» (Опубликовать)
4. Действия с публикациями:
   — «❤️» — лайк (toggle)
   — «✏️» — редактирование (для авторов и модераторов)
   — «🗑️» — удаление
5. Групповой чат: доступен из раздела сообщества для общения с одногруппниками.


3.2.15 Профиль

1. В боковом меню выберите «Профиль» или нажмите аватар вверху.
2. Данные профиля:
   — Имя, email, телефон, дата рождения, город, адрес
   — Редактирование: нажмите «Өзгерту» (Изменить)
3. Аватар: нажмите на фото для загрузки нового.
4. Настройки:
   — Язык интерфейса: RU / KZ / EN
   — Тема: светлая / тёмная
5. Баланс Q-coins, Premium-статус, достижения.
6. Выход: нажмите «Шығу» (Выйти) с подтверждением.


3.2.16 Техподдержка (для студентов)

1. В боковом меню выберите «Қолдау» (Поддержка).
2. Нажмите «Жаңа өтініш» (Новое обращение).
3. Опишите проблему (5-5000 символов).
4. Привяжите к курсу (опционально).
5. Нажмите «Жіберу» (Отправить).
6. Менеджеры получат уведомление и ответят.


3.2.17 Работа с платформой: Администратор

1. Войдите как admin@edu.kz / admin123.
2. Откроется административная панель с вкладками:

   ОБЗОР:
   — KPI-карточки: пользователи, студенты, курсы, преподаватели
   — Графики активности и регистраций
   — Быстрые ссылки

   ПОЛЬЗОВАТЕЛИ:
   — Таблица всех пользователей (поиск, фильтр по роли)
   — Создание: нажмите «+ Пользователь», заполните форму, выберите роль
   — Редактирование: нажмите на строку → всплывающая форма с полями для роли
   — Привязка родителя: выберите поле parent_id
   — Удаление: кнопка удаления с подтверждением
   — Вкладка «Очередь групп»: задачи добавления оплативших студентов в группы

   КУРСЫ:
   — Список курсов: создание, редактирование, активация/деактивация
   — Управление модулями и темами: добавление, порядок, контент, видео
   — Статистика курса: записи, прогресс, оценки

   ТЕСТЫ:
   — Привязка к курсу и теме
   — Добавление вопросов (20 шт.): текст, 4 варианта, правильный ответ
   — Локализация на казахский язык

   АНАЛИТИКА:
   — Графики: прогресс студентов, распределение оценок, популярность курсов
   — Фильтры по периоду и курсу

   ЗАЯВКИ:
   — Список заявок на курсы: pending, paid, approved, rejected
   — Одобрение/отклонение с данными студента и родителя

   ОТЗЫВЫ:
   — Модерация отзывов: одобрение, отклонение, удаление

   МАГАЗИН:
   — Управление товарами: создание, редактирование, деактивация
   — Заказы: просмотр всех заказов, статусы, назначение курьеров

   ПОДДЕРЖКА:
   — Все обращения студентов
   — Ответ (staff_note), закрытие тикета


3.2.18 Работа с платформой: Преподаватель

1. Войдите как teacher1@edu.kz / teacher123.
2. Откроется дашборд преподавателя.

   ГРУППЫ:
   — Список ваших учебных групп
   — Создание группы: нажмите «+ Группа», выберите курс, введите название
   — Добавление студентов: по email или из списка
   — Приглашение со-преподавателя: нажмите «Пригласить учителя»

   ЗАДАНИЯ (стиль Google Classroom):
   — Нажмите «+ Задание» для создания
   — Заполните: название, описание (Rich Text — форматирование, списки, код)
   — Прикрепите файлы (перетаскивание), ссылки, видео
   — Настройте рубрику: добавьте критерии с баллами
   — Установите дедлайн и настройки (закрытие после срока, разрешение редактирования)
   — Сохраните — задание появится в ленте группы

   ПРОВЕРКА РАБОТ:
   — В разделе «Журнал» (Gradebook) выберите группу и задание
   — Список студентов со статусами: сдано, не сдано, оценено
   — Нажмите на студента: просмотрите работу, выставьте баллы по рубрике
   — Напишите приватный комментарий (опционально)
   — Нажмите «Бағалау» (Оценить)

   МАТЕРИАЛЫ И ВОПРОСЫ:
   — Добавляйте учебные материалы и вопросы к темам курса через соответствующие разделы Classroom.

3.2.19 Аналитика и прогресс (Студент)

1. В боковом меню выберите «Аналитика».
2. Обзор активности:
   — График пройденных тем за неделю.
   — Количество выполненных заданий и средний балл.
   — Сравнение вашей активности с предыдущей неделей (%).
3. Дерево навыков: визуализация компетенций по курсу (программирование, логика, дизайн и др.).
4. Рекомендации ИИ: персональные советы на основе ваших слабых мест в тестах.

3.2.20 Достижения и ежедневные квесты

1. На дашборде найдите виджет «Күнделікті тапсырмалар» (Ежедневные квесты).
2. Выполняйте задачи: вход в систему, прохождение урока, сдача теста.
3. При заполнении шкалы квеста нажмите «Сыйлықты алу» (Забрать награду). Q-coins будут начислены мгновенно.
4. В профиле откройте раздел «Жетістіктер» (Достижения) для просмотра полученных бейджей (например, «First Win», «Active Week»).

3.2.21 Управление Premium-подпиской

1. В боковом меню выберите «Premium».
2. Ознакомьтесь с преимуществами: х2 монеты, скидки в магазине, отсутствие лимитов на видео.
